## 의사결정 흐름

### 1단계 — 과제 분석 및 코드 초안 생성 (Claude)

<sub>약 30~40분 소요</sub>

Claude에게 다음 순서로 요청해 전체 프로젝트 구조의 큰 틀을 잡았습니다.

1. **레포 구조 설계**: "디렉토리 구조를 제안해달라"고 요청한 뒤 직접 검토하고 보완했습니다.
(사진)
2. **코드 초안 생성 및 수정 반복**: 각 모듈(`ingest`, `transform`, `extractor`, `validator`, `db`, `models`, `slack_payload`, `prompt`, `run_pipeline`)의 초안을 생성하고, 정상 실행될 때까지 수정하였습니다.
3. **DB 스키마 검토**: Claude와 ChatGPT에 스키마 초안을 각각 던져 피드백을 받고 두 의견을 비교해 보완하였습니다.

### 2단계 — 코드 검수 및 클리닝 (ChatGPT)

<sub>약 4~5시간 소요</sub>

코드 전수 검사를 수행하였습니다. 불필요한 코드와 모듈을 제거하고, 프롬프트 엔지니어링을 보완하며, 파이프라인 전반의 코드 구성을 개선하는 작업을 수행하였습니다.

### 3단계 — 목데이터 생성 (ChatGPT)

<sub>약 30분 소요</sub>

제공된 JSON과 동일한 스키마를 유지하되 내용은 다른 목데이터 4건을 생성했습니다. 광고주명, 회의 주제, 발화 내용이 자연스럽게 다양화되도록 프롬프트를 구성했습니다.

```bash
#프롬프트 예시
회의 1 광고주: 노바드림 날짜: 2025-05-28 액션아이템: 8개 분량
회의 2 광고주: 노바드림 날짜: 2025-06-04 액션아이템: 10개 분량
회의 3 (짧게) 광고주: 노바드림 날짜: 2025-06-11 액션아이템: 6개 분량

위 결정 사항을 바탕으로 JSON스키마를 따라 목데이터를 생성........(생략)
```

### 4단계 — 대시보드 구현 (Gemini)

<sub> 약 1시간 소요</sub>

Streamlit 대시보드 위젯 구성과 구현 방법을 Gemini와 논의했습니다. 시각화 항목 선정 후 다크 테마 디자인 config를 직접 작성해 넘겨주고 Gemini가 이를 반영한 코드를 생성하면 검토·수정하는 방식으로 진행했습니다.

```bash
#프롬프트 예시
기존 대시보드를 “Toss 스타일의 분석 UI”로 리디자인해줘.

- 레퍼런스: Toss(토스) 앱 / 토스 증권 / 토스 데이터 대시보드 UI
    - 차분한 다크 테마 기반
    - 높은 가독성 + 정보 밀도 유지

- 컬러
    - Base Background: #101826 (Toss Dark Navy / Slate 계열)
    - Surface(Card):#1F2937 + 투명도 레이어 구조
    - Primary Accent: #0064FF (Toss Blue)
    - Secondary Accent: Blue → Purple 그라데이션 (analytics highlight용)
    - 상태 컬러: Success / Warning / Danger는 채도를 낮춘 muted tone 사용

- 시각화 스타일
    - Line chart: spline 기반 smooth curve 적용
    - Area chart: gradient glow 적용 (bottom → top fade)

- 폰트
    - SF Pro Display (macOS 느낌)

```

## AI 결과물을 직접 수정한 주요 판단 사례

### 1. 광고주명 기본값 제거 → 필수 인자로 변경

AI가 생성한 초기 설계에서는 advertiser 필드가 "노바드림"으로 고정된 기본값을 가지도록 되어 있었습니다. 

광고주는 회의마다 변경되는 핵심 식별자이기 때문에 기본값으로 하드코딩될 경우 필터링 오류 + advertiser와 title은 회의 데이터의 정체성을 결정하는 핵심 메타 정보이므로, 선택 입력이 아니라 생성 시 반드시 명시적으로 전달되어야 한다고 판단

=> --advertiser와 --title을 CLI의 필수 인자로 변경하여, 데이터 누락 가능성을 구조적으로 차단하도록 하였습니다.

### 2. 시스템 프롬프트 구체화

AI가 생성한 초안의 프롬프트에서 한국어 특성을 반영해 다음 항목을 추가했습니다.

- 액션아이템 판별 유형 3가지 명시 (명시적 업무 지시 / 자발적 의사 표현 / 합의·결정 표현)
- confidence 기준 4단계 정의 (담당자·기한 명확도 기준)
- 단순 의견, 문제 제기, 현황 공유는 추출 제외 명시

### 3. 대시보드 구성 조정

대시보드 초안은 단순 테이블 나열이었습니다.
=> 실제 운영 맥락에서 의미 있는 뷰를 만들기 위해 아래 위젯을 추가하였습니다.

- 주차별 회의·액션아이템 발생 추이 (스프라인 곡선 + 면적 강조)
- 담당자별 미완료 워크로드 + 드릴다운 테이블
- 반복 검출 키워드 Top 15
- Confidence 연속 분포 히스토그램 + 슬라이더 기반 검토 항목 필터링
