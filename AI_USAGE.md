# AI_USAGE.md

이 문서는 프로젝트 전반에서 AI 도구를 어떻게 활용했는지, 어떤 판단으로 결과물을 수정했는지를 기록합니다.

---

## 사용 도구

| 도구 | 주요 용도 |
| --- | --- |
| Claude | 코드 초안 생성, 시스템 아키텍처 다이어그램, 데이터베이스 스키마 설계 |
| Gemini | Streamlit 대시보드 디자인 및 위젯 구현 |
| ChatGPT | Claude 생성 초안 피드백 및 코드 검수, 목데이터 생성 |

---

## 의사결정 흐름

### 1단계 — 과제 분석 및 코드 초안 생성 (Claude) · 약 30분

Claude에게 다음 순서로 요청해 전체 구조의 큰 틀을 잡았습니다.

1. **레포 구조 설계**: "광고 에이전시 회의록에서 액션아이템을 추출하는 파이프라인을 만들어야 한다. 모듈 분리 기준과 디렉토리 구조를 제안해달라"고 요청한 뒤 직접 검토하고 보완했습니다.
2. **코드 초안 생성 및 반복 수정**: 각 모듈(`ingest`, `transform`, `extractor`, `validator`)의 초안을 생성하고, 정상 실행될 때까지 오류 메시지를 붙여 피드백을 반복했습니다.
3. **DB 스키마 검토**: Claude와 ChatGPT에 동일한 스키마 초안을 각각 던져 피드백을 받고 두 의견을 비교해 최종 결정했습니다.

### 2단계 — 코드 검수 및 클리닝 (ChatGPT) · 약 4시간

전체 파일을 ChatGPT와 함께 검토했습니다. 불필요한 코드와 모듈을 제거하고, 프롬프트 엔지니어링을 보완하며, 파이프라인 전반의 효율을 개선하는 작업을 논의 형태로 진행했습니다. 전체 과정 중 가장 많은 시간을 투입한 단계입니다.

### 3단계 — 목데이터 생성 (ChatGPT) · 약 30분

제공된 JSON과 동일한 스키마를 유지하되 내용은 다른 목데이터 4건을 생성했습니다. 광고주명, 회의 주제, 발화 내용이 자연스럽게 다양화되도록 프롬프트를 구성했습니다.

### 4단계 — 대시보드 구현 (Gemini) · 약 1시간

Streamlit 대시보드 위젯 구성과 구현 방법을 Gemini와 논의했습니다. 시각화 항목 선정 후 다크 테마 디자인 config를 직접 작성해 넘겨주고 Gemini가 이를 반영한 코드를 생성하면 검토·수정하는 방식으로 진행했습니다.

## 작업 방식에 대하여

일정이 촉박한 만큼 AI에게 구현 전체를 맡기기보다 **방향성 지시와 의미 부여에 집중**했습니다. 어떤 모듈이 필요한지, 왜 이 구조여야 하는지, 어떤 필드가 있어야 하는지를 직접 판단하고 AI는 그 판단을 코드로 빠르게 구체화하는 역할을 맡겼습니다.

---

## AI 결과물을 직접 수정한 주요 판단 사례

### 1. 광고주명 기본값 제거 → 필수 인자로 변경

AI가 생성한 초안에서 `advertiser` 필드의 기본값이 `"노바드림"`으로 하드코딩되어 있었습니다. 광고주명은 회의록 분석과 대시보드 필터링의 핵심 식별자인데 기본값이 있으면 누락 상태로 데이터가 쌓일 수 있다고 판단했습니다. `--advertiser`와 `--title`을 CLI 필수 인자로 변경해 누락 자체를 방지했습니다.

### 2. 시스템 프롬프트 구체화

AI가 생성한 초안의 프롬프트는 "액션아이템을 추출하세요" 수준의 일반적인 지시였습니다. 광고 에이전시 도메인 특성을 반영해 다음 항목을 직접 추가했습니다.

- 액션아이템 판별 유형 3가지 명시 (명시적 업무 지시 / 자발적 의사 표현 / 합의·결정 표현)
- confidence 기준 4단계 정의 (담당자·기한 명확도 기준)
- 단순 의견, 문제 제기, 현황 공유는 추출 제외 명시
- 실제 발화 패턴 기반 few-shot 예시 1쌍 추가

### 3. 대시보드 구성 조정

Gemini가 생성한 초안은 단순 테이블 나열 수준이었습니다. 실제 운영 맥락에서 의미 있는 뷰를 만들기 위해 다음을 직접 기획하고 반영을 지시했습니다.

- 주차별 회의·액션아이템 발생 추이 (스프라인 곡선 + 면적 강조)
- 담당자별 미완료 워크로드 + 드릴다운 테이블
- 반복 검출 키워드 Top 15
- Confidence 연속 분포 히스토그램 + 슬라이더 기반 검토 항목 필터링
